# Code Of Conduct

RepoGrammar's community should be direct, technical, and respectful.

## Expected Behavior

- Keep discussion focused on evidence, reproducible behavior, and clear
  tradeoffs.
- Assume contributors are acting in good faith unless there is concrete
  evidence otherwise.
- Give actionable feedback tied to files, commands, logs, or documented
  requirements.
- Respect privacy and do not request or post secrets, private source, prompts,
  credentials, or proprietary repository details.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private information without permission.
- Pressuring maintainers or users to share private code or credentials.
- Spam, fake engagement, paid stars, star trading, or automated promotion.
- Repeatedly pushing unsupported product claims after maintainers have pointed
  to the documented boundary.

## Reporting

Report security issues through the repository's security advisory path described
in `SECURITY.md`. For conduct concerns, open a private maintainer contact path
if available, or use a GitHub issue only when the report does not expose private
information.

Maintainers may remove comments, close issues, block accounts, or report abuse
to GitHub when behavior harms contributors or users.
