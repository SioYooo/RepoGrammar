## Goal

Make RepoGrammar useful to Codex and Claude Code during normal implementation
work by steering agents to `repogrammar_context` before broad grep/find/read
loops, while preserving sound-by-abstention semantics and source-output safety.

This slice does not expand Python semantic analysis. It turns existing family
evidence and read plans into a safer context protocol for coding agents.

## Current MCP Surface

The default MCP surface is one read-only tool: `repogrammar_context`.

Supported operations remain:

- `find_analogues`
- `show_family`
- `explain_deviation`
- `check_conformance`

Current inputs include `operation`, `target`, `token_budget`, `mode`,
`include_variations`, `include_exceptions`, and `include_source_spans`.
`mode=compact|evidence|deep` controls metadata density only. Source text is not
returned by default; `include_source_spans: true` is the explicit source-output
opt-in and is not implied by `mode: deep`.

MCP `initialize` now returns server instructions that spell out agent-first
adoption rules (call `repogrammar_context` before grep/find/Read), the
fallback-to-Read cases (`UNKNOWN`, stale, omitted, or insufficient), and the
source-span opt-in contract. The `repogrammar_context` tool description carries
the same adoption guidance.

## Current Output Shape

Family output contains metadata fields for:

- classification and support;
- selected evidence metadata;
- `read_plan`;
- typed unknowns;
- conformance/check status.

The current `read_plan` item fields are:

- `purpose`
- `path`
- `content_hash`
- `start_byte`
- `end_byte`
- `start_line`
- `end_line`
- `estimated_tokens`
- `why`
- `source_required_before_edit`
- `source_snippets_included`

When `include_source_spans` is absent, line ranges remain `null` and
`source_snippets_included` is `false`. When it is requested, a bounded
source-span renderer (now wired into both CLI and MCP) reuses the hash-checked
`SourceStore` boundary, fills line ranges for rendered spans, and returns
line-numbered text under a separate `source_spans` block alongside a
`source_span_policy` describing fallback guidance. Stale, missing,
hash-mismatched, too-large, unsupported, dynamic, insufficient, or conflicting
spans are omitted with explicit Read/Grep fallback guidance rather than rendered
from stale evidence.

## Current Installer Surface

Live agent wiring is global-first and supports Codex and Claude Code through
their native MCP CLIs (`codex mcp add/remove`, `claude mcp add/remove --scope
user`). A CodeGraph-style target registry also recognizes Cursor, opencode,
Hermes, Gemini, Antigravity, and Kiro for dry-run and `--print-config` planning;
live MCP writes for those targets and all project-local instruction-file writes
remain deferred until each adapter has an ownership receipt, uninstall inverse,
and tests.

Managed instruction-file integration is the remaining adoption gap. The managed
writer uses the exact markers `<!-- BEGIN REPOGRAMMAR MANAGED SECTION -->` and
`<!-- END REPOGRAMMAR MANAGED SECTION -->`, creates/appends/replaces the managed
section idempotently, removes only that section on uninstall, refuses malformed
partial markers, writes atomically with re-read verification, and records the
instruction path and action in the receipt without recording any source, query,
prompt, or path-evidence. Where a target's real instruction-file path is not
verified, a resolver plus env override and tests are implemented but live
writing stays deferred for that target rather than guessing a path. The writer
must preserve unrelated user content and must never apply RepoGrammar's own
mirrored guide policy to consumer repositories.

## Current Telemetry and Experiment Support

Telemetry is opt-in and source-free. Existing policy forbids uploading source
text, repo names, paths, content hashes, byte ranges, raw targets, symbols,
query text, evidence text, prompts, diffs, patches, environment variables, or
credentials.

Read displacement metrics may be added only as aggregate counters under that
policy. Token-savings claims require paired experiments and must not be inferred
from estimated read-plan token costs alone.

## UNKNOWN and Fallback Rules

Source spans must not be rendered from stale or unsupported evidence. The
renderer should preserve these cases as typed `UNKNOWN` or source-span omission:

- `StaleEvidence`
- `InsufficientSupport`
- `DynamicImport`
- `RuntimeDependencyInjection`
- `PytestFixtureInjection`
- `FrameworkMagic`
- `ConflictingFacts`
- source file missing, too large, non-UTF-8, or content-hash mismatch

Some unknowns block only specific subclaims. For example, a FastAPI dependency
target `UNKNOWN` may require reading dependency details but should not
necessarily erase route-family membership if the route claim itself has exact
fresh support.

## Review and Test Cadence

Do not run the full test suite after every helper edit. Finish a coherent slice,
review the logic for stale evidence, source leakage, path safety, deterministic
ordering, and false confidence, then run targeted tests and the required full
gate.
