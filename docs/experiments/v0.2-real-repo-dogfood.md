# v0.2 Real-Repo Dogfood Protocol

This protocol defines private dogfood before broader public preview messaging.
It is a runbook, not evidence that token savings have been measured.

## Repository Set

Run on five local repositories that can be analyzed without downloading code
during the experiment:

1. small Express API;
2. medium Express API;
3. Vitest-heavy library;
4. Jest app;
5. mixed JS/TS monorepo.

Do not commit private repository paths, source snippets, prompts, credentials,
or raw diffs to RepoGrammar.

## Commands

For each repository:

```text
repogrammar init
repogrammar index --json --progress never
repogrammar families --json
repogrammar find --json <representative-target>
repogrammar find --json --include-source-spans <representative-target>
repogrammar check --json <representative-target>
repogrammar stats --json
```

## Metrics

Record only aggregate or sanitized metadata:

- discovered file count;
- family count by language/framework role;
- `UNKNOWN` count by reason code;
- false-positive audit count;
- false-negative notes;
- read-plan item count;
- source-span omission count;
- stale-evidence count;
- baseline Read/Grep tool-call count for a matched task;
- RepoGrammar-assisted Read/Grep tool-call count for the same task.

## Audit Questions

- Did any React, package-only, dynamic, raw, or unresolved TS/JS lookalike become
  a JS/TS family?
- Did any object-literal `.get`, custom test wrapper, dynamic receiver, dynamic
  method, or unresolved alias become support?
- Did default CLI/MCP output include source text?
- Did source-span opt-in render only bounded line-numbered spans?
- Did stale/hash-mismatched evidence return `UNKNOWN` or omission guidance?

## Reporting

Write a private dogfood report outside this repository if it contains private
repo names or paths. A sanitized summary may be committed only if it contains no
source snippets, repo names, absolute paths, symbols, prompts, diffs, hashes, or
credentials.

Do not claim measured token savings unless a paired baseline/treatment run has
comparable token counts, provider/model metadata, task outcome, and correctness
assessment.
