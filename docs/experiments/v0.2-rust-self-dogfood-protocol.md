# v0.2 Rust Self-Dogfood Protocol

Purpose: evaluate whether RepoGrammar can reduce repeated manual reads while
working on RepoGrammar's own Rust implementation patterns.

This protocol measures potential read displacement, not causal token savings.
Actual token-saving claims require paired baseline/treatment experiments with
comparable task correctness.

## Setup

1. Build and install the current `repogrammar` binary.
2. Run `repogrammar init` in the RepoGrammar checkout if needed.
3. Run `repogrammar index --progress always`.
4. Confirm `repogrammar families --json` includes only supported internal Rust
   self-dogfood families and no stale claims.

## Task Categories

- Add or modify indexing phases.
- Add or modify family claim gates.
- Add or modify parser adapters.
- Add or modify CLI/MCP handlers.
- Add or modify storage validation paths.
- Add or modify source-span/read-plan behavior.
- Add or modify product tests.

## Measurement

For each task, record:

- task name;
- baseline files manually read before using RepoGrammar;
- RepoGrammar family/read-plan output mode used: compact, evidence, or deep;
- `estimated_potential_token_savings` reported by the local metric path;
- whether the returned family was useful, stale, insufficient, or `UNKNOWN`;
- whether source-span opt-in was needed;
- final validation commands.

Do not report measured token savings unless a paired baseline/treatment run with
comparable task correctness exists. If RepoGrammar returns `UNKNOWN`, stale
evidence, or an irrelevant family, record that result instead of forcing a
positive claim.

## Stop Conditions

- A Rust family claim appears for unsupported generic Rust code.
- A macro/cfg/trait-dispatch UNKNOWN is hidden or guessed away.
- Default output includes source snippets.
- A query result leaks absolute paths.
- Indexing requires Cargo/rustc/build-script execution.
