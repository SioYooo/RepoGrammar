- Date: 2026-06-27
- Branch: `feat/v0-2-public-preview-support-readiness`
- Status: conditionally ready for a bounded public preview after local
  validation. Published prerelease artifacts, npm publication, and a private
  dogfood pass remain pending.

## Support Matrix

| Area | Public-preview status | Evidence boundary |
|---|---|---|
| Python FastAPI | Supported | Bounded exact-anchor family evidence with typed `UNKNOWN` for dynamic, stale, or unresolved claims. |
| Python pytest | Supported | Bounded test/fixture family evidence; ambiguous fixture injection stays `UNKNOWN`. |
| Python Pydantic | Supported | Bounded model/settings family evidence; dynamic factories and unresolved bases stay `UNKNOWN`. |
| Python SQLAlchemy | Supported | Bounded model/repository evidence; dynamic declarative behavior remains conservative. |
| JS/TS Express | Conservative v0.2 preview | Exact import/require binding plus direct literal route calls; support>=3 and complete-link compatibility required. |
| JS/TS Jest/Vitest | Conservative v0.2 preview | Imported runner, imported alias runner, or ambient test-file runner with safe project context; support>=3 required. |
| React, Next.js, Fastify, Prisma, Drizzle | Not supported | Must not form public JS/TS families; current fixtures verify no derived support/family claim for representative lookalikes, and React-shaped TypeScript semantic-worker facts are blocked. |
| Full JS/TS semantic analysis | Not supported | No compiler-backed TypeScript resolution, complete alias/re-export semantics, project execution, or dynamic wrapper support. |
| Source text output | Explicit opt-in only | Default CLI/MCP output is metadata-only; source spans require explicit opt-in and hash/freshness checks. |
| Token savings | Not claimed by default | Only paired baseline/treatment experiments may report measured savings. |

## JS/TS Support Cases

- `express_exact_routes`: TypeScript Express routes with exact app/router
  bindings form a family, while object-literal lookalikes, reassigned/shadowed
  receivers, dynamic receiver calls, and dynamic methods do not derive support.
- `javascript_exact_routes`: JavaScript Express routes form a family under the
  same exact-anchor gates.
- `jest_vitest_exact_tests`: imported and aliased Vitest/Jest-style runners form
  suite/test families; custom wrappers do not derive support.
- `jest_vitest_ambient_context_tests`: ambient test-file runners can form
  families only with safe package dependency or JSON config test-runner context;
  script configs are metadata/typed `UNKNOWN` only and non-test-file ambient
  calls remain unsupported.
- `javascript_jest_vitest_exact_tests`: JavaScript imported, aliased, and
  ambient-with-context Jest/Vitest cases form JavaScript test families.

## Unsupported and Deferred Cases

- React/Next/Fastify/Prisma/Drizzle are public-preview unsupported. The
  `unsupported_framework_lookalikes` fixture records dependency/config context
  but must not produce derived support or family rows.
- React components/hooks, Next pages, Fastify routes, Prisma repositories, and
  Drizzle repositories require future framework-specific exact-anchor designs
  before they can become public claims.
- TypeScript compiler semantics, full path-alias/re-export resolution, dynamic
  wrappers, runtime DI, project execution, and project-local live install remain
  deferred.

## Known Risks

- Release workflow matrix is defined but has not been proven by a real tag run
  in this report.
- Preview install documentation uses an explicit tag such as
  `v0.2.0-preview.0`; relying on GitHub's `latest` redirect remains unsafe for
  prerelease artifacts.
- npm publication is not yet proven; local `npm pack --dry-run` validates package
  shape only.
- npm cached binary reuse currently checks for the cached binary and bundled
  worker file before execution; cached checksum metadata and manifest
  revalidation remain future hardening.
- Unsupported JS/TS framework fixtures are representative lookalikes, not broad
  framework test suites.
- TS/JS support remains exact-anchor and structural; false negatives are
  expected for wrappers, dynamic routing, deep barrel exports, and compiler-only
  binding cases.
- The current line-based TS/JS syntax scanner can over-report typed `UNKNOWN`
  for strings or comments that contain dynamic-import-shaped text; this is
  conservative noise, not support evidence.

## Blockers Before Public Preview

- Publish a prerelease tag and verify that all release artifacts and installer
  script checksum files are present.
- Publish or dry-run the npm package from the tag context.
- Run at least one private dogfood pass using
  `docs/experiments/v0.2-real-repo-dogfood.md` before making any token-saving
  marketing claim.

## Recommended Public Release Wording

RepoGrammar is a pre-alpha, local-first public preview. Python FastAPI, pytest,
Pydantic, and SQLAlchemy support is bounded and sound-by-abstention. JS/TS
support is a conservative v0.2 preview for Express route handlers and
Jest/Vitest suites/tests under exact-anchor evidence only. Unsupported,
dynamic, stale, ambiguous, or low-support cases return typed `UNKNOWN`. Default
output is metadata-only; bounded source spans require explicit opt-in.
