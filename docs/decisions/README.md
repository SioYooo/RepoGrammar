# Architecture Decision Records

Accepted ADRs capture durable decisions that are more stable than implementation
notes. Each ADR includes status, date, context, decision, alternatives,
consequences, and follow-up work.

When an ADR conflicts with lower-priority documentation, update the lower
priority document or write a superseding ADR.

Current accepted ADRs:

- ADR-0001: Rust core and local binary.
- ADR-0002: SQLite and FTS5 local index.
- ADR-0003: Pattern-family repository representation.
- ADR-0004: Rust core with language-native semantic workers.
- ADR-0005: TypeScript/JavaScript-first MVP with Python as second language.
