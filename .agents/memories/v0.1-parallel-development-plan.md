## Durable knowledge

- The current implemented foundation includes repo-local lifecycle, TS/JS
  discovery, syntax-only code-unit extraction, generation-scoped SQLite activation,
  active `files`/`units` reads for file-manifest-only or syntax-only inventory,
  syntax-origin framework-role fact storage for recognized Express, React, and
  Jest/Vitest code-unit shapes, semantic-fact/evidence storage substrate,
  internal active claim-input snapshot reads, and Rust-side TypeScript
  semantic-worker request/output protocol plus process validation. `index`/`sync`
  also support opt-in semantic-worker fact ingestion through
  `REPOGRAMMAR_TYPESCRIPT_WORKER` plus optional
  `REPOGRAMMAR_TYPESCRIPT_WORKER_ARGS_JSON`. The query application layer can run
  an internal file-hash freshness/readiness gate over snapshot semantic facts.
  Stored facts become EC-MVFI-lite family evidence only when they are strong,
  same-generation, fresh, and compatible with the framework role; arbitrary
  semantic facts and syntax-origin `FRAMEWORK_HEURISTIC` facts stay blocked as
  insufficient support.
- Do not assign agents to build storage from scratch or add first indexing;
  those slices exist and should be extended through canonical specs.
- Parallel implementation should split by boundary: Python parser/import graph,
  framework-role extraction, usage-driven propagation, target-centered call
  recovery, pytest fixture graph, EC-MVFI-lite family-claim gates,
  repository/worktree freshness metadata, storage read path, family-query
  contracts, MCP transport, installer writes, docs/tests.
- Keep ownership disjoint by path and canonical document. Integrate only after
  reviewing logic and running the required gates.
- Major features need a dedicated branch and atomic commits with tests/docs.
- The completed substrate now includes conservative family query reads and
  read-only MCP serving plus narrow global Codex/Claude MCP installer writes.
  It also includes a v0.1 TS/JS release fixture smoke gate for product CLI JSON
  paths over committed Express, React, Jest/Vitest, mixed JS/TS, low-support,
  and mixed-language fixture shapes. Query reads exact-match `family` and
  `member` targets, gate public family detail against current source hashes,
  and keep advisory check/conformance contexts as `CONTEXT_ONLY` with nested
  `UNKNOWN`. ADR-0011 now makes Python-first analysis the official v0.1 target,
  so new language-analysis work should follow
  `docs/plans/python-v0.1-implementation-plan.md` instead of extending the
  transitional TS/JS substrate as the primary product path.

## Revalidation conditions

Update after Python parser/import/framework analysis, Tree-sitter integration,
TypeScript compiler worker source, full family-claim gates, installer writes,
stable MCP API support, or any phase-scope change.
