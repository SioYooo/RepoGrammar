## Durable knowledge

- The substrate hardening and read-only MCP serving slices are complete enough
  for the current branch. ADR-0011 pivots the next official v0.1 implementation
  step to Python-first analysis planning and then Python parser/import/framework
  evidence, not deeper TS/JS family evidence by default. CodeGraph runtime
  integration and Tree-sitter replacement remain deferred.
- The hardening checkpoint is anchored to a source-backed audit of
  `c078a82cdfefbdb436adb6df80ac871bd6be615a`, but agents must re-inspect
  current HEAD before editing because later commits may already fix or change
  individual items.
- P1 hardening covers generation immutability, lifecycle doctor hygiene, and
  repo-guard coverage for current planning/specification/memory artifacts.
- P2 hardening covers bounded filesystem reads, parent Git ignore behavior,
  JSON-based bootstrap manifest validation, split manifest/storage schema
  reporting, semantic-worker request-size alignment, and the index-lock release
  gate.
- The index-lock release gate now has a narrow implementation: `index` and
  `sync` acquire `.repogrammar/locks/index.lock` before generation preparation,
  hold it through validation and activation, refuse live or unknown ownership,
  replace confirmed stale same-host locks during acquisition, clean up their own
  successful lock, and report lock state through `doctor`.
- Later v0.1 family work should resume with EC-MVFI-lite grouping, family-claim
  builders that can consume stored syntax-origin framework-role facts alongside
  stronger evidence, and evidence-backed `families`/`family`/`find`/`check`/
  `explain` output or typed `UNKNOWN`.
- The FamilyStore storage substrate now exists for generation-scoped family
  records, members, variation slots, and family-bound evidence. It is not yet
  populated by `index`/`sync`, and pattern-family CLI/MCP queries still require
  EC-MVFI-lite claim builders and family-bound evidence before they can become
  implemented. Syntax-origin framework-role facts are stored now, but remain
  framework-heuristic inputs only.
